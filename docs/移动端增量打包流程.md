# 移动端增量打包流程

移动端打包使用输入指纹和产物完整性校验。没有影响当前阶段的文件变化，并且已有产物完整时，直接复用产物并跳过命令执行。

## 缓存层级

```text
移动端源码、依赖、Vite 配置
          │
          ▼
Web dist ────────────── 未变化时跳过 vue-tsc 和 Vite
          │
          ▼
Capacitor 原生同步 ──── 未变化时跳过 cap sync
          │
          ├── Android Debug APK
          ├── Android Release APK
          ├── iOS Simulator App（仅 macOS）
          └── iOS Release Archive（仅 macOS）
```

每层缓存同时校验：

- 输入文件路径、内容和 SHA-256；
- 依赖锁文件和相关工作区包源码；
- 输出文件或输出目录的文件数量、总大小和 SHA-256；
- Android Release 的正式签名密钥和签名配置。

源码不变但产物缺失、被修改或缓存记录损坏时，会自动重新构建，不会复用不完整产物。

缓存记录保存在：

```text
.release-local/mobile-build-cache/
```

该目录已被 Git 忽略。

## 常用命令

构建 Android Debug 和 Release：

```powershell
npm run mobile:package:android
```

只构建正式签名 APK：

```powershell
npm run mobile:package:android:release
```

只构建 Debug APK：

```powershell
npm run mobile:package:android:debug
```

只构建 Web 或同步原生工程：

```powershell
npm run mobile:web:build
npm run mobile:native:sync
```

macOS 上构建 iOS 模拟器 App 或 Release Archive：

```bash
npm run mobile:package:ios:simulator
npm run mobile:package:ios:archive
```

iOS Archive 需要 Xcode、CocoaPods、Apple Team 和有效签名。Windows 会明确拒绝执行 iOS 原生打包，不会生成伪产物。

## 强制重建

Android Debug 和 Release 全量重建：

```powershell
npm run mobile:package:force
```

任意模式也可以直接追加 `--force`，或设置 `MOBILE_FORCE_REBUILD=1`。

只清除增量缓存记录和 iOS DerivedData，不删除 Android APK：

```powershell
npm run mobile:package:cache:clear
```

## 输出说明

日志中出现以下前缀时表示：

```text
[复用] 输入和产物均未变化，跳过该阶段
[构建] 输入或产物发生变化，执行该阶段
[完成] 阶段已成功并写入缓存记录
[失败] 阶段失败，不写入可复用记录
```

Android Release 在首次重新构建时还会执行正式签名验证。后续复用前会重新计算 APK SHA-256；只要 APK 被修改，就会自动失效并重建、验签。
