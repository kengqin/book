# 移动端测试环境目录

Windows 本机的 Android 测试环境统一放在 D 盘，不使用 C 盘的 Android SDK、AVD 或 Gradle 缓存。

```text
D:\NovelLibrary\mobile-test-env\
├── Android\
│   ├── Sdk\                    # Android SDK、platform-tools、emulator
│   └── Avd\                    # NovelLibraryApi35.avd
└── Gradle\                     # Gradle wrapper、依赖和构建缓存
```

当前用户环境变量：

```text
ANDROID_HOME=D:\NovelLibrary\mobile-test-env\Android\Sdk
ANDROID_SDK_ROOT=D:\NovelLibrary\mobile-test-env\Android\Sdk
ANDROID_AVD_HOME=D:\NovelLibrary\mobile-test-env\Android\Avd
GRADLE_USER_HOME=D:\NovelLibrary\mobile-test-env\Gradle
```

项目 Android 工程的本地 SDK 配置位于 `apps/mobile/android/local.properties`，同样指向 D 盘。

## 启动 Android 模拟器

```powershell
$sdk = 'D:\NovelLibrary\mobile-test-env\Android\Sdk'
$env:ANDROID_HOME = $sdk
$env:ANDROID_SDK_ROOT = $sdk
$env:ANDROID_AVD_HOME = 'D:\NovelLibrary\mobile-test-env\Android\Avd'
& "$sdk\emulator\emulator.exe" -avd NovelLibraryApi35 -gpu swiftshader_indirect
```

ADB 路径：

```text
D:\NovelLibrary\mobile-test-env\Android\Sdk\platform-tools\adb.exe
```

## Android 构建

```powershell
$env:ANDROID_HOME = 'D:\NovelLibrary\mobile-test-env\Android\Sdk'
$env:ANDROID_SDK_ROOT = $env:ANDROID_HOME
$env:ANDROID_AVD_HOME = 'D:\NovelLibrary\mobile-test-env\Android\Avd'
$env:GRADLE_USER_HOME = 'D:\NovelLibrary\mobile-test-env\Gradle'
Set-Location apps/mobile/android
.\gradlew.bat assembleDebug
```

## iOS 说明

Windows 本机没有 Xcode、iOS Simulator、CocoaPods 或 IPA 运行环境。iOS 原生工程保留在：

```text
apps/mobile/ios/App
```

需要在 Mac 上安装 Xcode 后，用 `App.xcworkspace` 运行模拟器、真机或 TestFlight 流程。
