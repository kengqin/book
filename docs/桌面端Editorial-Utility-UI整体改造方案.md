# 小说书库桌面端 Editorial Utility UI 整体改造方案

> 状态：已审核，可实施  
> 方案版本：1.0  
> 审核日期：2026-07-19  
> 当前产品基线：NovelLibrary v0.5.6（`develop`）  
> 视觉参考：NBchitu/CodexThemes-App `88d57eb2da08038049d0fce0993676115b7a6844`

## 1. 结论

本次改造采用“视觉体系整体移植、业务组件 Vue 化”的路线：保留现有 Vue 3、Vue Router、Tauri 2、Rust、本地数据库、更新中心、阅读器、笔记和 IDE 集成，只重构应用外壳、设计 token、通用交互组件与页面表现。

参考项目的主要价值是 Editorial Utility 设计规范、自定义 CSS token、侧边栏、页面标题、卡片、详情操作面板、设置分组和反馈模式。参考项目虽然依赖 Tailwind CSS 4，但实际页面主要使用集中式语义 CSS；Base UI React 也只负责 Dialog/AlertDialog。因此当前项目不需要迁移 React，也不需要引入 Tailwind 或大型 Vue UI 框架。

## 2. 目标与非目标

### 2.1 目标

- 建立可复用的浅色、深色、跟随系统三态设计系统。
- 统一侧边栏、页面标题、按钮、表单、卡片、面板、进度、空状态、Toast 和确认对话框。
- 将书架改造成适合竖版小说封面的响应式卡片网格。
- 重构书籍详情为“封面与说明 + 独立操作区 + 章节目录”。
- 统一搜索、工具、IDE 集成、设置、更新中心和笔记的视觉语言。
- 保持阅读器白色、纸张、夜间三套独立阅读主题，不受应用外观污染。
- 保持最小窗口 760×560 可用，900px 以下折叠侧边栏。
- 保留现有业务行为、数据格式和性能优化。

### 2.2 非目标

- 不迁移 React、Zustand、Base UI React。
- 不引入 Tailwind、Pinia、Element Plus、Naive UI 或 Ant Design Vue。
- 不复制 Codex Themes 名称、Logo、主题图片、截图或其他品牌资产。
- 不改变本地数据库 schema、导入解析协议、阅读进度结构或更新清单格式。
- 不在第一阶段实现在线书城、云同步或新的内容功能。
- 不伪造 macOS 红黄绿窗口按钮；窗口装饰继续由 Tauri/系统负责。

## 3. 基线复核与必须保留的近期改动

实施基于 v0.5.6，以下近期能力属于约束条件，UI 改造不得回退：

1. **书架摘要与缓存**：`LibraryView.vue` 已直接使用 `DesktopBookSummary`，`list_book_summaries` 避免逐书读取章节。书籍卡片必须直接消费摘要，不得恢复 `listDesktopChapters()` N+1 查询。
2. **书架即时呈现**：缓存存在时先显示旧结果，再后台刷新。重构模板时保留 `loading` 与 `refreshing` 的区分。
3. **更新清单缓存**：更新历史会先呈现内置/缓存清单，再后台获取远端清单。新 UI 不得恢复阻塞式空白加载。
4. **更新状态来源**：继续显示 `remote`、`cached`、`local` 三种来源和刷新状态。
5. **启动等待与启动页**：保留 `wait_for_startup`、20 秒上限、至少 900ms 展示和新版品牌启动页。启动页是临时过渡层，不纳入应用 chrome 的“禁用渐变”约束。
6. **质量门禁**：必须通过 `npm test`、`npm run desktop:web:build`、`npm run plugins:validate`、Rust 格式检查和 Rust 测试。

## 4. 架构决策

### 4.1 技术映射

| 参考项目 | 小说书库实现 |
| --- | --- |
| React 19 | Vue 3.5 |
| Zustand | Vue composable/module state |
| Lucide React | `lucide-vue-next` |
| Base UI Dialog | Vue 封装的原生 `<dialog>` |
| Tailwind 4 + 语义 CSS | 原生 CSS token + 语义类 |
| React 页面组件 | Vue SFC |
| Tauri 2 | 保持现有 Tauri 2 |

### 4.2 CSS 分层

保留 `src/styles.css` 作为入口，逐步拆分：

```text
apps/desktop/src/styles/
├── tokens.css
├── base.css
├── shell.css
├── components.css
├── library.css
├── book.css
├── search.css
├── tools.css
├── settings.css
├── notes.css
└── reader.css
```

迁移期间可以先在现有 `styles.css` 底部加入新 token 和覆盖规则，待所有页面稳定后再做机械拆分，避免一次性移动大量选择器造成回归。

### 4.3 全局状态

新增 `composables/useAppearance.ts`，模块级响应式状态管理 `system | light | dark`。使用 `novel-library:appearance` 持久化，并监听 `prefers-color-scheme`。不为主题状态引入 Pinia。

反馈不额外引入全局 Toast 队列。成功、错误和处理中状态显示在操作发生区域；笔记编辑器使用顶部提示条，应用更新使用专用全局浮层，避免无上下文的小字和重复通知。

## 7. 组件规划

```text
apps/desktop/src/components/ui/
├── AppSidebar.vue
├── PageHeader.vue
└── UiConfirmDialog.vue

apps/desktop/src/components/library/
└── BookCard.vue
```

只抽取确实跨页面复用且包含行为的组件，避免把简单标签过度组件化。按钮、图标按钮、下拉框、搜索框、复选框、提示条、错误条、进度条、空状态、浮层菜单和面板使用 `editorial.css` 中的语义类统一实现。

### 7.1 基础控件规则

- 按钮完整覆盖主要、次要、文本、图标、危险、禁用、hover、active 和 `focus-visible` 状态。
- 下拉框移除平台默认箭头，统一高度、边框、圆角、箭头、hover、focus、disabled 与深色主题。
- 搜索框由外层容器持有边框与焦点环，内部输入框无二次描边；清空按钮有独立 hover 状态。
- 复选框统一未选、选中、键盘焦点与禁用状态，并在深色主题保持足够对比度。
- `UiConfirmDialog` 统一普通确认与危险确认；`CloseBehaviorDialog` 保留专用业务结构但共享遮罩、面板、按钮和复选框视觉。
- 页面内提示使用成功、错误和处理中语义样式；笔记提示条、全局更新浮层和空状态共享同一 token。
- 浮层菜单统一边框、圆角、阴影、选项高度、hover 与键盘焦点。
- 所有基础控件必须同时通过浅色、深色、760px 和 1180px 视口检查。

### 7.2 Dialog 规则

`UiConfirmDialog` 封装原生 `<dialog>`，负责自动聚焦、Escape、遮罩、标题/说明语义和操作区。视图层不再使用 `window.confirm`；删除书籍、删除笔记、目录/数据库切换、升级备份确认和历史版本降级确认均使用显式 Dialog。

## 8. 页面实施规格

### 8.1 应用外壳

- `App.vue` 保留 Tauri 事件和更新初始化，只拆出 `AppSidebar`。
- 选中项使用浅绿色圆角背景和绿色图标/文字。
- 设置项保留更新圆点。
- `GlobalUpdateStatus`、`CloseBehaviorDialog` 使用新 token 重绘，不改事件行为。

### 8.2 书架

- 继续使用 `DesktopBookSummary[]` 和缓存优先加载。
- 顶部保留刷新与导入；刷新只显示旋转状态，不清空现有书籍。
- 主体改为 2:3 封面网格，宽屏 4–5 列，中屏 3 列，窄屏 2 列。
- 卡片展示书名、作者、格式、章节数、字数、阅读进度。
- 卡片点击进入详情；继续阅读作为低权重快捷操作；删除移动到详情或更多菜单。
- 封面卡片使用 `object-fit: cover`，详情使用 `contain`。

### 8.3 书籍详情

- 改为封面、信息和右侧操作面板三段布局。
- 右侧面板包含开始/继续阅读、进度、当前章节和删除。
- 书籍主题色只用于小型强调，不染色整个应用页面。
- 章节目录保持原逻辑，改为细边框、清晰 hover 的两列/单列响应式列表。

### 8.4 搜索

- 保留全文搜索 service。
- 标准 PageHeader + 最大 780px 搜索工作区。
- 增加搜索错误显示；搜索中、无结果、初始状态分别呈现。
- 结果使用紧凑列表，保留书籍、章节类型和摘要。

### 8.5 工具与 IDE 集成

- 工具首页使用两列大型功能卡。
- IDE 页按“检测 → 选择目标 → 安装/更新”组织状态。
- 不改变插件清单、检测与安装命令。

### 8.6 设置

分组为常规、外观、数据与存储、备份、更新。每行左侧标题/说明，右侧控件/动作；长路径支持换行和复制/打开。新增外观选择，但不改变存储迁移、备份和关闭行为。

### 8.7 更新中心

- 初始立即显示缓存/内置历史。
- 顶部版本状态面板包含当前版本、最新版本、检查/下载/安装动作和进度。
- 历史记录继续显示远端/缓存/内置来源及后台同步状态。
- 保留兼容性提示、降级警告、自动检查、后台检查和自动下载。

### 8.8 笔记

- 保持双栏编辑器、TipTap 状态和存储逻辑。
- 只统一 token、工具栏、菜单、选中态和反馈。
- 删除改为 `UiDialog`；正文继续使用衬线字体。

### 8.9 阅读器

- 白、纸、夜三套阅读主题独立定义，不继承管理界面 surface。
- 工具栏统一控件尺寸和焦点样式。
- 保留字号、行距、紧凑窗口、快捷键和进度保存。
- 沉浸布局放到第二阶段，第一阶段不改变滚动根节点，避免进度回归。

## 10. 实施阶段

1. **基线与 token**：保存现状、建立颜色/尺寸/字体/focus/reduced-motion。
2. **基础组件与外壳**：Appearance、Sidebar、PageHeader、按钮、Panel、Dialog、Toast。
3. **核心内容页**：书架、书籍详情、搜索。
4. **管理页**：工具、IDE、设置、更新中心。
5. **复杂工作区**：笔记与阅读器视觉统一。
6. **清理与验证**：删除旧规则、视觉检查、测试、构建、许可。

当前实施范围覆盖阶段 1–6；最终以真实 Tauri 窗口、浅色/深色和宽窄视口复核结果为准。
