# 小说书库阅读器

小说书库阅读器是“小说书库”桌面端面向 VS Code 系列编辑器的配套扩展。它可以在开发环境中浏览本地书架、章节和正文，并将阅读位置与桌面端保持同步。

正文通过编辑器装饰以只读方式显示，不会写入或修改当前源文件，也不会改变项目内容。

## 核心功能

- 在活动栏阅读面板中浏览书架、章节和当前五行正文。
- 在代码编辑器中使用“段落模式”或紧凑的“行尾模式”显示内容。
- 固定显示当前章序、章名与全书总进度；章节切换可使用快捷键或阅读面板标题栏。
- 逐行滚动至章节边界时自动切换到相邻章节，并一次展示新章节的多行文字。
- 自动同步书籍、章节和行位置；重新打开编辑器后可从上次位置继续。
- 独立保存代码内阅读的显示状态，隐藏后翻行或切章不会强制重新显示。
- 支持从当前编辑器导入 TXT 或 EPUB 文件，并可快速打开小说书库桌面端。

## 支持的编辑器

同一个 VSIX 可用于以下编辑器：

- Visual Studio Code 与 VS Code Insiders
- Cursor、Trae、Qoder、Windsurf、Kiro
- VSCodium、Void、Code - OSS、Positron、PearAI

要求编辑器兼容 VS Code `1.85.0` 或更高版本的扩展 API。不同编辑器版本的菜单位置和图标样式可能略有差异。

## 开始使用

1. 安装并启动“小说书库”桌面端，导入至少一本 TXT 或 EPUB 书籍。
2. 安装本扩展后重启编辑器，或执行 `Developer: Reload Window`。
3. 点击活动栏中的“小说书库”图标，选择书籍和章节。
4. 使用阅读面板标题栏、正文区域鼠标滚轮或快捷键控制阅读。

插件通过桌面端提供的本机 Bridge 读取书架并同步进度。若桌面端尚未运行，插件会在其启动后自动尝试重新连接。

## 默认快捷键

| 功能 | Windows / Linux |
| --- | --- |
| 显示或隐藏代码内阅读 | `Ctrl+Alt+N` |
| 切换段落模式 / 行尾模式 | `Ctrl+Alt+9` |
| 上一行 / 下一行 | `Ctrl+Alt+↑` / `Ctrl+Alt+↓` |
| 上一章 / 下一章 | `Ctrl+Alt+←` / `Ctrl+Alt+→` |
| 打开小说书库桌面端 | `Ctrl+Alt+D` |

点击阅读面板标题栏中的键盘图标，可查看快捷键并进入当前编辑器的 Keyboard Shortcuts 设置。用户自定义绑定始终优先于默认值。

## 实验性增强滚轮

VS Code 扩展 API 不向代码装饰开放鼠标滚轮和点击事件。小说书库桌面端因此为每个已检测到的 VS Code 系列编辑器提供独立的“增强滚轮”开关，并保持默认关闭。

用户明确启用后，桌面端会先备份 Monaco 工作台文件，再安装带完整性校验的交互桥接：

- 鼠标位于小说正文装饰区域时，滚轮切换上一行或下一行。
- 鼠标移出小说区域后，文件仍按编辑器默认行为滚动。

关闭开关会恢复备份；编辑器升级覆盖工作台后可能需要重新启用。任何写入步骤失败时，桌面端都会自动回滚。普通 VSIX 安装不会自行启用此功能。

## 数据与隐私

- 插件不直接访问或修改小说书库数据库。
- 插件不修改源文件，正文仅作为临时的只读编辑器装饰显示。
- 书架读取和进度同步仅通过回环地址上的本机 Bridge 完成。
- Bridge 使用桌面端运行时生成的临时端口与令牌，不向局域网或公网提供服务。

## 故障排查

- 未显示书架：确认桌面端正在运行，然后点击阅读面板中的“刷新书架”。
- 安装后没有入口：重启编辑器，或执行 `Developer: Reload Window`。
- 增强滚轮未生效：完全退出并重新打开对应编辑器；编辑器升级后请在桌面端重新启用。
- 快捷键冲突：打开 Keyboard Shortcuts，搜索 `小说书库` 或 `@ext:novel-library.novel-library-reader` 后重新绑定。
