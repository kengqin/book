# 更新日志

本项目遵循 [Semantic Versioning](https://semver.org/lang/zh-CN/)。桌面端正式版本与 Git Tag、GitHub Release、安装包版本保持一致。

## [Unreleased]

## [0.6.5] - 2026-07-24

### 新增

- 三套 IDE 插件新增固定章节信息行，统一展示当前章序、章名和全书总进度；滚动越过章节边界时自动衔接上一章或下一章，并直接展示新章节的多行正文。
- 桌面端扩展 Code OSS 类编辑器支持范围，覆盖 VS Code、VS Code Insiders、Cursor、Trae、Qoder、Windsurf、Kiro、VSCodium、Void、Code - OSS、Positron 和 PearAI。
- Code OSS 类编辑器新增默认关闭的“增强滚轮”实验开关，提供工作台备份、完整性校验、关闭恢复和失败自动回滚。

### 优化

- 三套 IDE 插件统一升级为 `0.4.15`，统一使用桌面端“书本 + 书签”品牌图标，并重写正式、规范的插件市场说明和使用文档。
- 桌面端插件管理的成功、失败和操作结果统一使用全局 Message 提示，移除页面内提示框与侧边强调条，并优化提示层级和质感。
- 暂时隐藏代码编辑器固定首行中的“上一章 / 下一章”文字按钮，保留快捷键、滚轮阅读和章节边界自动衔接。
- Code OSS 类编辑器统一使用各自官方 CLI 安装、更新、卸载并复检同一个 VSIX；移除会遮挡正文的小说装饰悬浮提示。

### 修复

- 修复切换小说、关闭后重新打开或快速连续滚动时阅读位置回到开头的问题；进度写入改为顺序执行，切书前排空待写入任务并重新读取桌面端最新位置。
- 修复 JetBrains“自定义快捷键”未进入 Keymap 对应动作的问题，并完善运行中 IDE 的安装、更新和卸载提示及正常关闭流程。
- 修复插件卸载错误未通过全局 Message 提示的问题，并增强 Code OSS 分支编辑器 CLI 不兼容时的扩展目录清理与结果复检。
- 修复桌面端调试构建被 `target/debug` 中旧插件清单覆盖、导致本地启动仍显示旧插件名称和版本的问题。

### 升级说明

- 桌面端数据库 Schema 保持为 6，现有书库、阅读进度、笔记和自定义数据目录保持不变，无需备份即可升级。

## [0.6.0] - 2026-07-19

- 设置页新增“常规 / 数据 / 更新”分组导航，配置项按使用场景集中管理。
- 新增统一的圆角下拉菜单和开关组件，选中状态、焦点状态和关闭行为保持一致。

### 优化

- 设置页统一采用浅灰背景、白色圆角面板和薄荷绿强调色，减少无意义辅助文字，突出状态与操作。

## [0.5.6] - 2026-07-19

### 修复

## [0.5.0] - 2026-07-17

### 更新与发布

- 发布流程拆分为暂存、Windows 构建、校验文件和最终发布四个阶段，并在失败时保护上一版 `latest.json`。
- 新增 Windows NSIS 固定版本 Manifest、公开 Release 端到端验证、SHA256SUMS 和历史版本不可改写校验。
- 更新中心统一检查、下载、取消、安装和重启状态，补充签名、版本、官方地址与下载长度校验。
- 新增默认关闭的后台定时检查和后台自动下载设置；自动下载后仍由用户确认安装和重启。
- 桌面端与 IDE 插件版本改为独立维护；统一使用各 IDE 官方安装命令，彻底移除直接打开 `cli.js` 的路径。
- 默认数据迁移到安装目录内的 `NovelLibraryData`，旧 C 盘受管数据会在校验迁移成功后清理，后续切换数据位置会同步迁移并清理旧数据库。

### 版本管理

- 桌面端版本只代表桌面应用发布；VS Code/Cursor、JetBrains 和 Visual Studio 插件分别维护自己的版本。当前三个插件制品均为 `0.4.5`，以后桌面端发布不再要求同步升级插件。
- 更新迁移、插件定位、安装复检和卸载复检能力已纳入同一版本发布记录。

## [0.4.0] - 2026-07-17

### 新增

- 桌面工具库新增“小说阅读插件”，默认展示支持的 IDE，并允许用户选择安装、更新或卸载目标。
- 为 Visual Studio Code、Cursor、IntelliJ IDEA 和 Visual Studio 提供随桌面端发布的阅读插件。
- IDE 插件支持浏览书架和章节、在编辑器中紧凑阅读、逐行切换、自动换行以及开启或关闭阅读。
- 关闭桌面窗口时可选择每次询问、缩小到系统托盘或直接退出，并可记住上次选择。

### 优化

- 插件管理页展示插件名称、版本、安装状态和可执行操作，IDE 检测在后台静默完成。
- 统一各 IDE 的插件图标和阅读交互，优化行尾与段落阅读模式以及快捷键行为。
- 设置入口固定在侧栏底部且不显示选中底色，版本更新入口统一收纳到设置页。

### 修复

- 修复插件安装包缺失、检测超时、卸载后状态未刷新以及安装或卸载时意外打开 IDE 的问题。
- 修复当前书籍或章节暂不可用时阅读请求失败的问题，并改善桌面端未启动时的恢复提示。
- 修复部分 IDE 中阅读侧栏不显示、快捷键无效以及切换阅读行时错误打开工具窗口的问题。

### 升级说明

- 数据库 Schema 保持为 `5`，已有书库、阅读进度、笔记和自定义数据目录无需迁移。
- 已安装旧版 IDE 插件时，可在桌面端“工具”中的小说阅读插件页面更新。

## [0.3.2] - 2026-07-16

### 修复

- 修复新版本已出现在历史记录中，但顶部状态仍错误显示“当前已是最新版本”的问题。
- 更新检查改为读取版本清单后直接校验对应版本的固定更新元数据，避免 GitHub 最新版本地址的短暂缓存不同步。
- 在线版本记录增加缓存破除；更新元数据尚在同步时明确显示“新版已发布”，不再给出相反状态。

### 兼容

- 数据库 Schema 升级至 `5`，已有 EPUB 会自动补齐章节类型、原始章号并清理重复卷首内容；TXT、书库、阅读进度、笔记和自定义数据目录不受影响。
- 下载进度、取消下载、安装并重启和历史版本安装流程保持不变。

## [0.3.1] - 2026-07-16

### 修复

- 修复 EPUB 分卷标题页被错误归入“正文”分组的问题，已导入书籍会在启动时自动整理。

## [0.3.0] - 2026-07-16

### 新增

- 本地书库新增 EPUB2 / EPUB3 导入，支持元数据、封面、书脊顺序、嵌套目录和分卷识别。
- EPUB 正文支持标题、段落、列表、引用、代码、表格、插图和脚注锚点排版。
- EPUB 使用独立 Web Worker 解析，超长书籍导入时持续回报进度，不阻塞主界面。

### 安全

- EPUB 先解析为安全结构化内容，再由阅读器使用 DOMPurify 白名单进行最终清理。
- 从备份或数据库读取的富文本会移除脚本、事件属性、iframe、危险链接和第三方样式。

### 数据

- 数据库 Schema 升级至 `4`，新增来源格式、封面、正文格式和纯文本搜索索引。
- Schema 3 数据库和 v1 / v2 备份自动兼容，已有 TXT 书籍无需重新导入。

## [0.2.0] - 2026-07-16

### 新增

- 新增工具库模块，主导航进入工具库后再选择具体工具。
- 新增本地笔记工具，支持新建、编辑、搜索、置顶、复制和删除。
- 新增基于 Tiptap 的富文本编辑器，支持标题、粗体、斜体、下划线、列表、引用、代码块、链接、分隔线和撤销重做。
- 笔记支持自动保存、单篇 Markdown / HTML / JSON 导出，以及全部笔记 JSON 导入导出。
- 完整数据备份增加本地笔记，数据库 Schema 升级至 `3`。

## [0.1.2] - 2026-07-16

### 新增

- 新增跨页面全局更新状态，下载进度在书架、阅读器和设置页面持续显示。
- 更新下载支持取消；下载完成后由用户点击“安装并重启”，不再自动强制重启。

### 修复

- 修复 v0.1.1 更新器对象被 Vue 响应式代理后，点击更新出现 `Cannot read private member from an object whose class did not declare it` 并导致无法下载安装的问题。
- 修复历史版本 Release 和安装包链接无法通过系统浏览器打开的问题。
- 更新服务不可用时改为显示中文状态，不再暴露底层英文错误。
- 修复发布流程缺少 `latest.json`，导致自动更新检查返回 `404` 的问题。

### 升级说明

- v0.1.1 内置更新器存在兼容问题；应用内更新失败时，请下载 v0.1.2 安装包覆盖安装，书库数据不会被删除。

## [0.1.0] - 2026-07-15

### 新增

- 新增 Windows 桌面端，支持自定义安装目录。
- 新增 TXT 导入、编码识别、章节解析、本地书架和 SQLite 持久化。
- 新增书籍详情、分卷目录、章节阅读、全文搜索和阅读进度恢复。

- 新增 JSON 备份恢复、自定义数据目录和自定义 SQLite 数据库文件。

- 当前数据库 schema 为 `2`；安装旧版本前应先导出备份。
